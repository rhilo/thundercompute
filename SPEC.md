# SPEC.md — FLUX.1-dev LoRA Training Pipeline Orchestrator

## 1. Project Overview & Architecture
The objective is to build an automated, high-efficiency, production-grade LoRA training pipeline specifically targeting human subject assets for FLUX.1-dev. The system lifecycle is split between local asset staging and execution on a high-performance Thunder Compute remote instance (tnr-0), minimizing VRAM friction and pipeline overhead.

### Pipeline Stages
1. Stage 1 (Setup): Automated instance provisioner handling environment state, virtual environments, and custom CUDA dependencies.
2. Stage 2 (Preprocessing): Human-centric, face-aware anatomical aspect ratio cropping, image quality filtering, and duplicate elimination.
3. Stage 3 (Captioning): Vision-Language structural captioning using JoyCaption Alpha Two optimized for execution in bfloat16.
4. Stage 4 (Config Orchestration): Exact YAML synthesis matching the strict schema of ai-toolkit without manual intervention.

## 3. Component Specifications

### 3.1. setup.sh (The Environment Provisioner)
An idempotent, clean bash wrapper designed to establish a completely reproducible remote execution environment.
* Shell Enforcement: Must begin with set -euo pipefail.
* State Management: Detect if a Python virtual environment (.venv) exists. If not, instantiate it, upgrade pip, and ensure all downstream execution forces native activation.
* PyTorch Target: Force-install PyTorch compiled specifically for CUDA 12.8 to optimize modern cloud execution:
  pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu128 -q
* Git Idempotency: When cloning ai-toolkit, if the directory already exists, perform a defensive git pull followed by a recursive submodule synchronization (git submodule update --init --recursive) rather than skipping entirely.

### 3.2. preprocess.py (The Intelligent Dataset Sanitizer)
A performance-focused image sanitation pipeline that strips noise, removes duplicates, and conforms assets to precise training targets.
* Target Dimensions: Images must resolve to exactly 768x1152 (2:3 Portrait orientation) using high-fidelity cv2.INTER_LANCZOS4 or Pillow equivalent interpolation, exported as high-quality JPEGs (95% quality matrix).
* Anatomical Face-Biasing: Refactor standard face detection into a rock-solid, lightweight facial boundary solver. If a face is detected, compute a modified bounding centroid: shift the crop center downward by exactly 15% of the calculated crop height. This guarantees the head rests perfectly inside the upper-third aesthetic quadrant of the target portrait canvas, preserving full shoulder/body lines.
* Quality & Deduplication Gates: 
  * Compute Laplacian variance (cv2.Laplacian().var()) to filter out micro-blur below a customizable threshold (--blur-threshold).
  * Compute perceptual hashes (imagehash.phash) for every surviving image. Reject incoming frames displaying a Hamming distance below the target delta (--hash-threshold) to discard semantic duplicates.

### 3.3. caption.py (The High-Fidelity JoyCaption Pipeline)
This component builds descriptive text alignments utilizing fancyfeast/llama-joycaption-alpha-two-hf-llava.
* VRAM Optimization: The LLava vision pipeline must load natively using explicit torch_dtype=torch.bfloat16 mapped directly to execution boundary device_map=0. 
* Token Integrity: Follow upstream directives meticulously to prevent double <bos> token injections through misaligned chat template applications.
* Prompt Architecture: Guide the engine via system/user blocks to systematically output pure descriptive prose focusing heavily on textiles, hair structures, stance, background environment, and lighting without inventing personal identification names.
* Synthesized Output: Capture output vectors, strip out runtime formatting text, and write directly to an adjoining .txt` file mirroring the base image name. Prepend the target trigger string natively: [trigger_word], [detailed descriptive prose].
* Caching: Skip files with existing caption files unless explicitly forced via --overwrite.

### 3.4. generate_config.py (The Schema Synthesizer)
Automates the structural construction of configuration files consumed directly by ai-toolkit.
* Deterministic Model Mapping: ai-toolkit requires a Hugging Face repo id (`black-forest-labs/FLUX.1-dev`) or a local diffusers directory (e.g. `/home/ubuntu/FLUX.1-dev` with `transformer/config.json`). A single ComfyUI `flux1-dev.safetensors` file is detected only for user guidance; it is not valid as `name_or_path`.
* Fallback Asset Scan: Prefer local diffusers folders, then HF cache snapshots, then the HF repo id. ComfyUI single-file paths are scanned to warn when only a monolithic checkpoint is present:
  1. /home/ubuntu/flux1-dev.safetensors
  2. ~/ComfyUI/models/unet/flux1-dev.safetensors
  3. ~/ComfyUI/models/diffusion_models/flux1-dev.safetensors
  4. ~/ComfyUI/models/checkpoints/flux1-dev.safetensors
* Precision Parameters: Enforce dtype: bf16 execution metrics for high dynamic range gradient tracking, saving final consolidated checkpoints as compact float16 layers.
* Hyper-parameters Matrix: Default configuration values explicitly against professional setups:
  * Network Topology: Rank 16 / Alpha 16.
  * Optimizer Stack: adamw8bit paired with a target learning rate of 1e-4.
  * Bucket Matrix: Explicit aspect ratio bucketing bound to portrait profiles ().
  * VRAM Mitigation: Force quantize: true on the core model loading configuration block to ensure execution safety inside standard cloud hardware thresholds.

## 4. Operational Workflow Target
Once generated by Cursor, the unified stack must link together smoothly with the following baseline syntax:

# 1. Initialize dependencies and environment (use bash; do not run: sh setup.sh)
bash setup.sh

# 2. Extract, sanitize, deduplicate, and composition-crop raw inputs
python3 preprocess.py --zip /home/ubuntu/raw_assets.zip --output /home/ubuntu/dataset/clean_v1

# 3. Process vision-language captions with optimized bfloat16 tensors
python3 caption.py --dataset /home/ubuntu/dataset/clean_v1 --trigger "glecca"

# 4. Generate the definitive training configuration pointing directly to our base weights
python3 generate_config.py --dataset /home/ubuntu/dataset/clean_v1 --output /home/ubuntu/training_run_01 --config-out ~/ai-toolkit/config/flux_lora_v1.yaml --trigger "glecca"
